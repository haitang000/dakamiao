# 打卡喵 🐾

通过无障碍服务自动打开钉钉并完成每日打卡的安卓工具。核心目标：**省事**，且**任何时候都能被你叫停**。

> ⚠️ 是否使用、是否符合公司规定，请你自行判断。本工具**不伪造定位、不伪造人脸**——遇到人脸/拍照会停下来交还给你本人。

## 它怎么工作

1. **定时触发**（`AlarmManager` 精确闹钟）或**手动一键**触发。
2. 打开钉钉（`com.alibaba.android.rimet`）。
3. 无障碍服务按你配置的「步骤」逐个在界面里**按文字找到按钮并点击**（不写死坐标，换机型也不怕）。读不到文字（如按钮是图片贴图）时，自动用 **OCR 兜底**（见下）。
4. 点完最后一步后自动识别结果：
   - 出现「打卡成功」等字样 → 通知你成功；
   - 出现「人脸/拍照」等字样 → **立即停止并震动提醒**，由你本人完成验证；
   - 都没识别到 → 提醒你手动核对。

## 状态一眼可见：屏幕边框跑马灯

自动操作全程，屏幕四边会显示一圈流光边框：

- 🔵 **蓝色流光绕圈** = 正在正常操作；
- 🔴 **变红** = 操作受阻（比如某一步的按钮迟迟找不到，可能是关键词填错或界面不对）。

边框只画在最外圈、中间透明，且**不拦截任何触摸**，不会挡住钉钉或自动点击。需要悬浮窗权限。

## OCR 兜底（应对图片按钮）

极端情况下按钮是纯图片、无障碍读不到文字。此时打卡喵会：**截屏**（`AccessibilityService.takeScreenshot`，静默无快门声）→ **离线文字识别**（ML Kit 中文，模型打包在 APK 里，**不联网**）→ 在识别到的文字位置派发点击。

- 只在无障碍常规查找失败时才启动，限频约 1.3 秒一次，不影响正常性能；
- 需 **安卓 11（API 30）及以上**（截屏 API 限制），低版本自动跳过；
- 可在设置 ③ 区用「启用 OCR 兜底」开关关闭。

## 四重中断（随时叫停）

- 屏幕上的**悬浮红色「停止」按钮**（全程可见）
- **通知栏**里的「停止」
- **音量下键**急停（仅打卡进行中拦截，平时不影响调音量）
- 每一步之间都会检查停止标志

## 首次使用

打开 App，按 ① 区依次授予权限：

| 权限 | 作用 | 必需 |
|---|---|---|
| 无障碍服务 | 读取钉钉界面、模拟点击 | ✅ 核心 |
| 悬浮窗 | 显示「停止」按钮 | 强烈建议 |
| 通知 | 显示进度与结果 | 建议 |
| 精确闹钟 | 定时准点 | 定时功能需要 |
| 忽略电池优化 | 防止定时被系统杀 | 建议 |

## 关键一步：把「步骤」改成你钉钉里的真实文案

不同公司钉钉打卡入口/按钮文字不一样。请**先在钉钉里手动走一遍打卡**，把每一步按钮上的字，一行一个填进 App 的「③ 打卡步骤」：

- **上班步骤** 默认：`考勤打卡` → `上班打卡`
- **下班步骤** 默认：`考勤打卡` → `下班打卡`

（例如你司入口在「工作台」里，就把第一行改成 `工作台`，再加一行 `考勤打卡`……以此类推。）

改完点「保存」，再用「④ 立即执行」测试，直到稳定。

## 重要限制

- **定时触发那一刻手机需要亮屏/可操作**。屏幕全黑或安全锁屏时，钉钉界面渲染不出来，点击会失败。建议把打卡时间设在你通常会用手机的时段，或让手机保持亮屏。
- 需要人脸/拍照的公司，自动化只能帮你走到弹脸前一步，最后一下由你本人完成。
- 不处理定位，默认你本人就在打卡范围内。

## 构建

```
./gradlew :app:assembleDebug
```
产物：`app/build/outputs/apk/debug/app-debug.apk`

## 代码结构

| 文件 | 职责 |
|---|---|
| `MainActivity.kt` | 设置界面、权限引导、手动触发 |
| `AutoClockAccessibilityService.kt` | 核心：打开钉钉、找按钮点击、识别结果、悬浮窗、通知、中断 |
| `AlarmScheduler.kt` | 精确闹钟排程 |
| `ClockAlarmReceiver.kt` | 闹钟触发、每日重排、周末跳过 |
| `BootReceiver.kt` | 开机重排闹钟 |
| `StopReceiver.kt` | 通知栏「停止」 |
| `Prefs.kt` | 全部配置读写 |
